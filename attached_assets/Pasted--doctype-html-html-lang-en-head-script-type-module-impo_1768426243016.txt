Ah, I see what’s going on. What you’ve posted is your HTML head plus some Vite/React dev runtime scripts—not the font itself.

If your WOFF2 font isn’t showing correctly, it’s because the browser is trying to load it via CSS, not directly in the HTML <script> or <link>. Vite will serve your fonts as static assets if you place them correctly, but you need to reference them in CSS.

Here’s what you should do:

1. Place your font
project-root/
├─ public/
│  ├─ fonts/
│  │  └─ Inter-Bold.woff2


Anything in public/ is served as-is, so /fonts/Inter-Bold.woff2 will resolve correctly.

2. Add CSS for WOFF2
@font-face {
  font-family: 'Inter';
  src: url('/fonts/Inter-Bold.woff2') format('woff2');
  font-weight: 700;
  font-style: normal;
  font-display: swap;
}


Then use it:

body {
  font-family: 'Inter', sans-serif;
}

3. Optional: Preload for faster rendering

Add this to your <head>:

<link rel="preload" href="/fonts/Inter-Bold.woff2" as="font" type="font/woff2" crossorigin>

4. Important Vite tip

If you were trying to import the WOFF2 file in a module like JS:

import InterBold from './fonts/Inter-Bold.woff2';


Vite will give you a URL string you can use in @font-face dynamically:

@font-face {
  font-family: 'Inter';
  src: url('<%= InterBold %>') format('woff2');
  font-weight: 700;
  font-style: normal;
}
